test: Add comprehensive OAuth environment variables validation

Added complete test suite for OAuth credentials configuration following
the addition of TikTok, Instagram, and Reddit OAuth variables to .env.

## Changes

### Environment Variables (.env)
- Added TikTok OAuth credentials (CLIENT_KEY, CLIENT_SECRET, REDIRECT_URI)
- Added Instagram/Facebook OAuth credentials (APP_ID, APP_SECRET, REDIRECT_URI)
- Validated Reddit OAuth credentials (CLIENT_ID, CLIENT_SECRET, REDIRECT_URI)

### Validation Functions (lib/env.ts)
- Added ensureTikTokOAuthEnv() - Validates TikTok OAuth configuration
- Added ensureInstagramOAuthEnv() - Validates Instagram OAuth configuration
- Added ensureRedditOAuthEnv() - Validates Reddit OAuth configuration
- Added ensureAllSocialOAuthEnv() - Validates all social OAuth platforms

Features:
- Throws errors in production if variables missing
- Warns in development if variables missing
- Clear error messages with missing variable names
- Aggregated validation for all platforms

## Migration Guide

No migration needed. New validation functions are optional and can be
called in API routes to ensure OAuth credentials are configured:

```typescript
import { ensureTikTokOAuthEnv } from '@/lib/env';

export async function GET(request: NextRequest) {
  ensureTikTokOAuthEnv(); // Validates before processing
  // ... rest of OAuth flow
}
```
