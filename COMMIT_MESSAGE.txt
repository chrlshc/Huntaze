fix: migrate to Auth.js v5 for Next.js 16 compatibility

Fixes #500-auth-error

## Problem
- NextAuth v4 incompatible with Next.js 16 + Turbopack
- 500 error during authentication: "Cannot read properties of undefined (reading 'custom')"
- openid-client module trying to access Node.js APIs in Edge runtime

## Solution
- Upgrade to Auth.js v5 (next-auth@5.0.0-beta.30)
- Create centralized auth configuration in auth.ts
- Simplify route handlers to use new v5 API
- Force Node.js runtime for all auth routes
- Add experimental serverActions config
