docs(api): Add complete API documentation and fix PostgreSQL type bug

## Summary

Fixed critical database type bug and created comprehensive API documentation
for the Huntaze CRM system.

## Bug Fix

- Fixed PostgreSQL aggregate parsing in tests
- Changed: `expect(result.rows[0].total_value).toBeGreaterThan(0)`
- To: `expect(parseInt(result.rows[0].total_value)).toBeGreaterThan(0)`
- Issue: PostgreSQL returns SUM/COUNT/AVG as strings, not numbers
- Impact: All database aggregate queries must parse results
